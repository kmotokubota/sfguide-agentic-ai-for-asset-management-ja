# SAMデモ - ミドルオフィスオペレーションシナリオ

ミドルオフィスオペレーションロール向けの完全なデモシナリオ。ステップバイステップの会話、期待される応答、データフローを含みます。

---

## ミドルオフィスオペレーション

### Middle Office Copilot - NAV計算＆決済監視

#### ビジネスコンテキスト設定

**ペルソナ**: Sarah、Simulated Asset Managementのミドルオフィスオペレーションマネージャー  
**ビジネス課題**: ミドルオフィスオペレーションチームは、複数のポートフォリオとカストディアンにまたがる取引決済、照合差異、NAV計算、コーポレートアクション、キャッシュ管理を監視する必要があります。手動監視は時間がかかり、エラーが発生しやすく、リアルタイムの例外アラートを提供できないため、オペレーション障害、NAVエラー、決済ペナルティのリスクがあります。  
**価値提案**: すべてのミドルオフィスプロセスのリアルタイム監視、自動例外検出と根本原因分析、インテリジェントな是正推奨を提供するAI駆動のオペレーションインテリジェンス。

**エージェント**: `middle_office_copilot`  
**利用可能データ**: 10ポートフォリオ、3カストディアン、日次決済アクティビティ、照合データ、NAV計算、コーポレートアクション、キャッシュポジション

#### デモフロー

**シーン設定**: 午後4時GMT、Sarahは18時の日次NAV計算前にすべてのミドルオフィス機能の本日のオペレーションステータスをレビューする必要があります。決済失敗、照合差異、または今夜のNAVに影響を与える可能性のある問題を迅速に特定して解決する必要があります。

##### ステップ1: 決済失敗監視
**ユーザー入力**: 
```
過去3営業日のすべての決済失敗を表示し、原因を理解するのを手伝ってください。
```

**使用ツール**:
- `middle_office_analyzer` (Cortex Analyst) - SAM_MIDDLE_OFFICE_VIEWから過去3日間のStatus='Failed'のFACT_TRADE_SETTLEMENTをクエリ

**期待される応答**:
- テーブル表示: 取引ID、証券、カウンターパーティ、決済金額、経過日数、ステータス、失敗理由
- 深刻度フラグ: 🚨 FAILED (>T+2日経過 - 重大)、⚠️ PENDING (T+2ウィンドウ内)
- 各失敗の根本原因分析（SSI不一致、証券不足など）
- ETAと担当者付きの具体的な是正ステップ
- 失敗決済の合計金額とカウンターパーティ内訳

**トーキングポイント**:
- **リアルタイムオペレーション監視**: すべてのカウンターパーティにまたがる決済ステータスの即座の可視性
- **自動根本原因分析**: AIが具体的な失敗理由（SSI不一致、カウンターパーティ問題）を特定
- **深刻度分類**: 経過日数と規制要件に基づく重大vs保留中の決済

##### ステップ2: 照合差異調査

**ユーザー入力**: 
```
すべてのポートフォリオについて本日の照合差異をまとめ、重要なものにフラグを立ててください。
```

**使用ツール**:
- `middle_office_analyzer` (Cortex Analyst) - SAM_MIDDLE_OFFICE_VIEWからStatus='Open'の差異をDifference DESCでクエリ

**期待される応答**:
- 全体的な照合ステータス: マッチ率%、タイプと深刻度別の差異数
- 重大な差異のテーブル: 差異タイプ、ポートフォリオ、証券、差異金額、ステータス、深刻度
- 深刻度分類:
  - 🚨 CRITICAL: ポジション差異 >£1M または NAVの>1%
  - ⚠️ HIGH: キャッシュ差異 >£100K または NAVの>0.1%
  - Medium: タイミング差異 <£100K
- 各重大差異の根本原因分析
- 解決タイムラインと必要なアクション

##### ステップ3: NAV計算ステータス

**ユーザー入力**: 
```
すべてのファンドにおける本日のNAV計算の状況は？調査が必要な異常はありますか？
```

**使用ツール**:
- `middle_office_analyzer` (Cortex Analyst) - SAM_MIDDLE_OFFICE_VIEWからポートフォリオ別の最新NAVをクエリ

**期待される応答**:
- NAV計算サマリー: 完了ファンド、保留レビュー、調査中
- ファンド別テーブル: ファンド名、NAV、日次変化%、ステータス、検出された異常
- 異常アラート:
  - ⚠️ ANOMALY DETECTED: 対応する市場変動なしの>2% NAV変化
- フラグされた異常の根本原因調査
- 承認推奨: どのNAVが承認可能か vs 調査が必要か

##### ステップ4: コーポレートアクション処理ステータス

**ユーザー入力**: 
```
今後5営業日のすべての保留中のコーポレートアクションを表示し、即座の処理が必要なものをハイライトしてください。
```

**使用ツール**:
- `middle_office_analyzer` (Cortex Analyst) - SAM_MIDDLE_OFFICE_VIEWから今後のアクションをクエリ
- `search_custodian_reports` (Cortex Search) - 保留中のコーポレートアクションのカストディアン通知を取得

**期待される応答**:
- テーブル: 証券、アクションタイプ、権利落ち日、支払日、インパクト金額、ステータス、優先度
- 優先度フラグ:
  - ⏰ DUE TODAY: 権利落ち日が本日、即座に処理必要
  - ⚠️ PENDING: 2日以内に期限、準備必要
  - 🔍 RESEARCH NEEDED: 調査が必要な複雑なアクション（スピンオフ、合併）
- 各アクションの処理推奨
- NAVとキャッシュポジションへの影響

#### シナリオラップアップ

**ビジネスインパクトサマリー**:
- **運用効率**: 自動例外検出により手動監視時間を70%削減
- **リスク削減**: 決済失敗、照合差異、NAV異常の重大化前のプロアクティブな特定
- **NAV精度**: 自動異常検出とコーポレートアクション追跡によるNAVエラー防止
- **規制コンプライアンス**: タイムリーな決済監視によりT+2規制コンプライアンスとペナルティ回避を確保

### Middle Office Copilot - 完全なオペレーションステータス（キャッチオール）

#### ビジネスコンテキスト設定

**ペルソナ**: Sarah、Simulated Asset Managementのミドルオフィスオペレーションマネージャー  
**ビジネス課題**: オペレーションマネージャーは、マネジメントコールの準備や緊急エスカレーションへの対応時に、単一のリクエストで完全なオペレーションステータスレポートが必要になることがあります。  
**価値提案**: Middle Office Copilotは、単一の包括的な質問からすべてのオペレーションドメインを分析し、ステップバイステップのガイダンスなしで完全なオペレーション状況を提供する完全な自律オーケストレーションを実証します。

##### ステップ1: PDF付き完全なオペレーションステータス（全機能）

**ユーザー入力**: 
```
すべての決済失敗と根本原因分析、解決ステータス付き照合差異、NAV計算異常、コーポレートアクション処理ステータス、すべてのポートフォリオとカストディアンにまたがるキャッシュポジションサマリーをカバーする完全なオペレーションステータスレポートをください。マネジメントコール用のプロフェッショナルなPDFを生成してください。
```

**使用ツール**:
- `middle_office_analyzer` (Cortex Analyst) - FACT_TRADE_SETTLEMENT、FACT_RECONCILIATION、FACT_NAV_DAILY、FACT_CORPORATE_ACTIONS、FACT_CASH_POSITIONSにまたがる包括的クエリ
- `pdf_generator` (Generic) - `internal`オーディエンス付きブランドPDF生成

**期待される応答**:
- **決済ステータス**: 深刻度フラグと根本原因付きの失敗/保留取引
- **照合サマリー**: ポートフォリオとカストディアン別のオープン差異と経過日数
- **NAVステータス**: 本日のNAV計算と異常または警告
- **コーポレートアクション**: NAV締め切り前に注意が必要な保留アクション
- **キャッシュポジション**: 通貨とカストディアン別サマリーと資金調達懸念
- **優先度マトリックス**: 緊急度とインパクト別のすべての問題の統合ビュー
- **推奨アクション**: 所有者とETA付きの具体的な是正ステップ
- **PDF生成**: Simulatedロゴ付きプロフェッショナルブランドPDF
